1. 首先在本门主要讲git的课程之上，我初步了解到了git基础的使用方法。
例如最基本的:

- git init

- git add
 
- git commit -m " " 

- git status
 
- git push

- git clone

- git log 查看版本

当然不仅如此，还有就是如何创建ssh密钥等知识。

2. 在课下，通过群里面发的视频，我找到了GeekHour的两小时git教程，与其他一些资料，进一步了解了git,工作区，暂存区，仓库三者之间的联系，本地仓库与远程仓库的关联，分支与main的关系。
不仅如此，还学到了其他的一些命令，如:

- git reset --hard

- git reset --soft

- git reset --mixed

- git diff 查看“工”与“存”差异

- git diff HEAD 查看“工”与“仓”差异

- git diff --cached 查看“存”与“仓”差异

- git checkout -b <分支名> 创建并快速切换到分支

- git push origin <分支名> 推送分支

- git merge

- git rebase 
(出现conflict:手动解决-> git add-> git rebase --continue /倘若搞砸可git rebase --abort放弃这次变基回到初始状态)

远程仓库也就是云端，在GitHub上面，可以是自己创建的也可以是fork别人的，通过git push和git pull 这两个简单的命令与本地仓库相互联系。若想要在本地上传一个文件至远程仓库，可以在仓库根目录那里新建一个文件，通过git add就可以让这个文件添加至暂存区，并且使得这个文件被跟踪，git commit -m "提交信息"可以使得在暂存区的文件被添加至本地仓库，最后便是git push了，将本地仓库同步至远程仓库。GitHub 是一个基于 Git 版本控制系统的在线代码托管平台，它不仅提供分布式版本控制，还提供了多种协作功能，使得软件开发者和其他参与者能够方便地共事，在此之中，分支的存在有着很大的益处。分支相当于一个main的轻量级指针，在修改项目代码时，每个修改代码的人可以有自己的分支并在自己的分支里面进行修改，与main隔离开来，分支的存在让开发人员可以同时修改代码而对项目不起到毁坏的作用。在平时使用GitHub时，远程仓库可以设置为私人的，用于储存自己的代码，也可以是公共的，开源的，用于分享自己的技术，开源项目等，当然为别人的仓库作出自己的贡献也是可以的。
