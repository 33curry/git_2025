#Git学习总结
git 的工作方式是能轻松满足多人工作的好工具，能更好的备份文件，整合文件。
git 的语法对git仓库的管理是很重要的，但是不得不说其内容真是多，让人难以记住。因此我只好专门写了一些笔记，在忘记怎么用时找出来查看。各种用法之下又有模式，还有复合使用等多种用法。例如，git stash 和git reset就有多种用法。我的学习方法就是在问题中学习，通过搜索解决方法的同时，学这多种用法，记下这些笔记。同时也学到一个重要的理解：HEAD作为一个指针指向提交，而分支类似一个载体承载HEAD和提交，所以在分离头指针状态下就是没有分支这个载体，所以创建新提交后可能会丢失。
当然，git 终究只是一个工具，我现在没有学会所有的语法，但是我认为也不需要学会所有的语法，git的语法中不同的语法能产生相同的效果，所以为达目的，其实只需要学会一种熟悉且好用的就行，不必像一门学科似的深究。所以，在之后的工作中若遇到git语法上的问题，也是像学习时一样，学会一种解决办法即可。对于工作，工作效率是很重要的，所以克隆下仓库是最简单的工作模式了，是工作的最佳选择，更新远程仓库的更新和文件推送都是很方便的。
