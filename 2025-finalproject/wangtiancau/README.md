
 学习总结

在这门课程中，我系统地学习了 **Linux 命令行、Git 版本控制以及 GitHub 协作流程** 的基础与实践应用。  
通过每一次实验和作业，我逐渐掌握了命令行的使用逻辑、Git 的核心命令及其背后的原理。

---

 一、Linux 命令行部分

我学习并掌握了以下常用命令的使用：

- `ls`：列出当前目录下的文件和文件夹  
- `cd`：切换目录，例如 `cd ..` 返回上一级  
- `pwd`：显示当前工作路径  
- `mkdir`：创建新文件夹  
- `touch`：创建新文件  
- `rm`：删除文件或文件夹  
- `cp`：复制文件，例如 `cp a.txt b.txt`  
- `mv`：移动或重命名文件  
- `cat`：查看文件内容  
- `grep`：查找匹配的字符串  
- `echo`：输出内容到终端或文件  
- `chmod`：修改文件权限  
- `find`：查找文件  
- `tar`：打包压缩与解压文件  

通过反复练习，我理解了命令行操作的强大灵活性，并熟悉了文件系统的结构与路径管理。

---

 二、Git 基础操作

在 Git 部分，我学习并掌握了以下命令的用法：

- `git init`：初始化本地仓库  
- `git clone`：克隆远程仓库到本地  
- `git add`：将文件添加到暂存区  
- `git commit -m "message"`：提交修改到本地仓库  
- `git status`：查看当前仓库状态  
- `git log`：查看提交历史  
- `git diff`：比较修改前后的差异  
- `git branch`：查看或创建分支  
- `git checkout`：切换分支  
- `git merge`：合并分支  
- `git pull`：从远程仓库获取最新代码  
- `git push`：将本地修改推送到远程仓库  
- `git revert` / `git reset`：回退版本  
- `git restore`：恢复文件到某次提交的状态  

这些命令让我理解了版本控制的意义，知道了如何在多人协作中安全地同步、修改、回滚代码。

三、GitHub 协作与 Pull Request

通过本次大作业，我第一次完整体验了：
- fork 教师仓库
- 在自己仓库中修改内容
- 创建 Pull Request (PR)
- 与上游仓库同步
- 查看合并检查项、请求审阅 (Request review)

我明白了 Pull Request 的作用，不仅是提交作业，更是团队协作、代码审查的标准方式。  
同时也学会了如何保持线性提交历史、避免 merge 冲突。
